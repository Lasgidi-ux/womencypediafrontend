# Fix 500 Internal Server Errors for Nomination and Story Submissions

## Problem Analysis
The frontend is receiving 500 Internal Server Errors when submitting nominations and stories to the Strapi CMS. These errors indicate server-side issues preventing the API requests from being processed successfully.

### Error Details
- `POST /api/nominations` returns 500 (nomination submissions)
- `POST /api/contributions` returns 500 (story submissions)

### Likely Causes
1. Missing or misconfigured content types in Strapi
2. Missing required fields in content type schemas
3. Broken relations to other content types
4. Database schema mismatches
5. Controller or lifecycle hook errors
6. Missing permissions or roles

## Investigation Steps

### 1. Check Strapi Content Types
**Location**: Strapi Admin Panel > Content-Types Builder

**Required Content Types**:
- `nomination` (singular) - for nomination submissions
- `contribution` (singular) - for story submissions

**If missing**: Create the content types with appropriate fields.

### 2. Verify Field Configurations

#### For `nomination` Content Type:
- `title` (Text, required)
- `type` (Enumeration: values=['nomination'], required)
- `nomineeName` (Text, required)
- `nomineeEra` (Text/Relation)
- `nomineeRegion` (Text/Relation)
- `nomineeCategory` (Text/Relation)
- `reason` (Rich Text, required)
- `sources` (JSON)
- `nominatorName` (Text, required)
- `nominatorEmail` (Email, required)
- `subjectName` (Text)
- `content` (Rich Text)
- `contactName` (Text)
- `contactEmail` (Email)
- `region` (Text)
- `status` (Enumeration: values=['pending', 'approved', 'rejected'])

#### For `contribution` Content Type:
- `title` (Text, required)
- `type` (Enumeration: values=['story'], required)
- `status` (Enumeration: values=['draft', 'pending'], required)
- `content` (Rich Text, required)
- `storyType` (Text)
- `subjectName` (Text, required)
- `relationship` (Text)
- `region` (Text, required)
- `theme` (Text)
- `lessons` (Text)
- `contactName` (Text, required)
- `contactEmail` (Email, required)
- `permissionGranted` (Boolean, required)
- `media` (Media relation, multiple files allowed)

### 3. Check Relations
Ensure any relation fields (nomineeEra, nomineeRegion, nomineeCategory for nominations) point to existing content types.

### 4. Verify Permissions
**Location**: Strapi Admin > Settings > Users & Permissions Plugin > Roles

- Ensure `Public` role has `create` permission for both content types
- Check if authentication is required (currently submissions work without login)

### 5. Database Schema
**Location**: Strapi Admin > Settings > Users & Permissions Plugin > Advanced Settings

- Ensure the database tables exist and match the content type schemas
- Check for any migration issues

### 6. Server Logs
Check Strapi server logs for detailed error messages when the 500 errors occur.

## Implementation Steps

### Phase 1: Content Type Creation/Fixes
1. Access Strapi admin panel
2. Navigate to Content-Types Builder
3. Create or modify `nomination` and `contribution` content types
4. Add all required fields with correct types
5. Save and restart Strapi if necessary

### Phase 2: Testing
1. Test nomination form submission
2. Test story form submission
3. Verify data is saved correctly in Strapi admin
4. Check that relations work properly

### Phase 3: Frontend Improvements (if needed)
- Enhance error handling to provide better user feedback
- Add retry mechanisms for transient server errors
- Implement offline submission queue if server is frequently unavailable

## Success Criteria
- Nomination form submits successfully (200 response)
- Story form submits successfully (200 response)
- Data appears correctly in Strapi admin
- No 500 errors in server logs
- Users receive appropriate success/error messages

## Rollback Plan
If changes break existing functionality:
- Revert content type changes in Strapi
- Restore from backup if database issues occur
- Frontend will continue to work with improved error handling
